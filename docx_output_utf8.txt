Employee Attrition (Project I)
Business Understanding
Tahapan ini memahami masalah, menentukan tujuan bisnis, menilai situasi ketersediaan sumber daya, dan menghasilkan rencana proyek, misal business understanding pada project ini berkaitan dengan employee attrition, lengkapnya bisa dilihat dibawah ini:
Perusahaan Jaya Jaya Maju menghadapi tingginya tingkat attrition karyawan, melebihi 10%, yang secara signifikan mengganggu stabilitas operasional dan efisiensi. Fenomena ini mengindikasikan adanya ketidakpuasan atau faktor pendorong lain yang menyebabkan karyawan memutuskan untuk meninggalkan perusahaan. Tingginya attrition rate ini berpotensi menyebabkan kerugian finansial akibat biaya rekrutmen dan pelatihan karyawan baru yang tinggi, penurunan produktivitas tim, serta hilangnya keahlian dan pengetahuan institusional. Oleh karena itu, perusahaan perlu secara mendesak mengidentifikasi akar penyebab dari tingginya attrition ini agar dapat merumuskan strategi retensi yang efektif. 
Data Understanding
Tahapan ini dilakukan untuk mengidentifikasi, menganalisis kumpulan data, memverifikasi kualitas data. Hal-hal yang bisa dilakukan antara lain seperti dibawah ini:
Overview Dataset (sumber dataset bisa diakses di [1], secara overview dataset bisa dicek berkaitan dengan jenisnya, jumlah datanya, kelengkapan, dan lain-lain)
Cek format dataset (Dataset berupa tabular)
Cek jumlah dataset (Dataset terdiri dari 35 kolom dan 1470 baris, detailnya dapat di cek menggunakan dataframe info)
Cek dan pahami tipe data (baik dari sisi bahasa pemrograman maupun statistika, misal integer dengan jenis data rasio, dan lain-lain)
Cek kelengkapan dataset (Kolom attrition memiliki data null / missing sekitar 28.03%)
Perform Exploration Data Analyst (EDA) dengan teknik statistika misal cek outlier, imbalance, tren, dan lain-lain. Untuk project ini, EDA yang bisa dilakukan antara lain cek korelasi antar attrition ke multi variabel (multivariate), atau ke satu variabel (univariate), dan lain-lain
Kaitkan hasil pemahaman data dengan pemahaman bisnis (misal, berdasarkan EDA dan analisis lainnya, didapati bahwa data attrition perlu dilengkapi sebelum diidentifikasi faktor yang mempengaruhi tingginya attrition)
Data Preparation
Tahapan ini dilakukan untuk membersihkan data, menentukan set data yang akan digunakan, atau mendapatkan atribut baru yang akan membantu pemrosesan, memformat data, dan lain-lain. Teknik preprocessing bisa ditentukan berdasarkan hasil EDA (data understanding) DAN sesuai kebutuhan Modeling, contoh: 
Mengatasi data yang missing atau null values, misal dengan menghapus sebagian data attrition jika diketahui missing data jumlahnya sangat sedikit, atau memisahkan dataset yang missing kemudian diimputasi atau diprediksi menggunakan teknik tertentu 
Melakukan balancing data dengan teknik tertentu jika diketahui data jauh tidak seimbang, 
Transformasi format nilai data, misal dari teks ke integer, dan lain-lain
Scaling data atau merubah skala data, misal skala 0-100 menjadi 0-1, untuk kebutuhan keseragaman data
Pemilihan fitur / kolom yang berpengaruh terhadap data yang akan diprediksi (feature selection)
Splitting data untuk kebutuhan modeling,
Dan lain-lain
Modeling
Tahapan ini ditujukan untuk membuat dan menilai berbagai model menggunakan beberapa metode yang berbeda berdasarkan masalah yang dihadapi. Dalam kasus ini, pemodelan diusulkan untuk memprediksi missing data pada attrition dan identifikasi faktor yang berpengaruh pada attrition. 
Evaluation
Tahapan ini mengevaluasi hasil, melakukan peninjauan, memutuskan model mana yang paling sesuai dengan kebutuhan bisnis atau masalah menggunakan teknik-teknik tertentu. Penting di dalam data science project, model hasil evaluasi (model bagus yang dipilih) perlu dieksport untuk kebutuhan deployment termasuk inferensi atau dijadikan pretrained model
Deployment
Tahapan ini berfokus pada penerapan model yang telah terpilih ke dalam lingkungan operasional sehingga dapat digunakan oleh end user maupun sistem bisnis. Pada tahap ini, model yang sudah dianggap sesuai dengan kebutuhan bisnis akan diintegrasikan ke dalam aplikasi, dashboard, atau layanan berbasis API. Berhubung projek ini berkaitan dengan analisis faktor yang mempengaruhi attrition, jadi tahapan deploy yang dilakukan adalah:
Ekspor Model – menyimpan model terlatih dalam format yang sesuai dengan library yang digunakan:

keras/tensorflow: model.save("keras_model")
pytorch: torch.save(model.state_dict(), "pytorch_model.pth")
Ekspor hasil prediksi/inference– menyimpan hasil prediksi (klasifikasi, regresi, klastering, atau forecasting) dalam format tertentu
CSV/XLS: untuk data tabular yang ringan, khususnya untuk kasus ini yang berhubungan dengan analisis dengan dashboard
Parquet: untuk data tabular yang berukuran big data
JSON: untuk kebutuhan API sistem aplikasi
WAV: umumnya untuk dataset yang berkaitan dengan suara
MP4/AVI: untuk dataset yang berkaitan dengan video
PNG/JPG/TIFF: untuk dataset yang berkaitan dengan gambar
NumPy Array (.npy / .npz) → untuk menyimpan hasil prediksi matriks piksel

Masukan code dibawah ini pada Dockerfile
# Base image
FROM python:3.12-slim
# Set working directory
WORKDIR /app
# Copy requirements
COPY requirements.txt .
# Install dependencies
RUN pip install --no-cache-dir -r requirements.txt
# Copy all files ke container
COPY . .
# Expose port untuk Railway
EXPOSE 8000
# Run Flask
CMD ["gunicorn", "-b", "0.0.0.0:8000", "app:app"]
